### 1.4 现有数据库表结构

```sql
-- 核心业务表
CREATE TABLE files (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  path TEXT NOT NULL,
  extension TEXT NOT NULL,
  size INTEGER DEFAULT 0,
  created_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  modified_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  accessed_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  hash TEXT DEFAULT '',
  is_hidden INTEGER DEFAULT 0,
  is_readonly INTEGER DEFAULT 0,
  is_system INTEGER DEFAULT 0,
  attributes TEXT DEFAULT '',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 文件内容索引
CREATE TABLE file_contents (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  file_id INTEGER NOT NULL,
  content TEXT,
  content_preview TEXT,
  indexed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (file_id) REFERENCES files(id) ON DELETE CASCADE
);

-- 搜索历史
CREATE TABLE search_history (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  query TEXT NOT NULL,
  search_type TEXT DEFAULT 'filename' CHECK(search_type IN ('filename', 'content')),
  result_count INTEGER DEFAULT 0,
  search_count INTEGER DEFAULT 1,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 文件打开方式配置
CREATE TABLE file_open_config (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  extension TEXT NOT NULL UNIQUE,
  open_method TEXT DEFAULT 'internal',
  internal_viewer TEXT DEFAULT NULL,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引排除规则
CREATE TABLE index_exclude_rules (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  rule_type TEXT NOT NULL DEFAULT 'directory',
  pattern TEXT NOT NULL,
  description TEXT DEFAULT '',
  is_regex INTEGER DEFAULT 0,
  is_enabled INTEGER DEFAULT 1,
  priority INTEGER DEFAULT 0,
  ...
);
```

**已有索引：**
- `idx_files_name`, `idx_files_extension`, `idx_files_path`
- `idx_files_modified_time`, `idx_files_created_time`
- `unique_file(path, name)` - 唯一约束
- `idx_search_history_query`, `idx_search_history_created_at`

### 1.5 现有问题与不足

#### 🔴 必须解决的问题
1. **缺少分类导航 UI** - 后端 API 已存在，但前端没有侧边栏或分类入口
2. **搜索语法简单** - 仅支持空格分隔的关键词，不支持 `ext:`、`size:` 等高级语法
3. **排序功能弱** - v-data-table 仅支持单列点击排序，无法多字段组合排序
4. **无收藏/书签功能** - 缺少 `file_favorites` 表和相关逻辑
5. **无最近访问记录** - 缺少 `file_access_history` 表

#### 🟡 可以优化的问题
6. **预览体验一般** - hover 触发有时不够优雅，移动端不适用
7. **结果展示单调** - 只有表格视图，无网格视图
8. **无法批量操作** - 不支持多选、批量删除/复制
9. **无虚拟滚动** - 大结果集可能导致性能问题
10. **移动端适配差** - 左右布局在窄屏下体验不佳

### 2.2 模块一：即时搜索增强（P0 - 必须）

**目标：从"需要回车"升级为"输入即搜索"，并支持高级语法**

#### 当前状态
- ✅ 已实现 debounce 300ms 自动搜索
- ✅ 支持文件类型筛选（image/document/code/video/audio/archive/executable）
- ✅ 支持大小范围筛选（minSize/maxSize，单位 MB）
- ❌ 不支持搜索语法（`ext:`, `size:`, `date:`, `path:`）
- ❌ 不支持通配符（`*`, `?`）
- ❌ 不支持正则表达式
- ❌ 搜索结果无高亮

#### 增强方案

| 功能点 | 描述 | 实现难度 | 优先级 |
|--------|------|----------|--------|
| **搜索语法解析器** | 解析 `ext:pdf size:>10mb` 格式 | 中 | P0 |
| **扩展名筛选** | `ext:pdf` 或 `*.js` | 低 | P0 |
| **大小筛选增强** | `size:>10mb`, `size:<1kb`, `size:1mb-100mb` | 低 | P0 |
| **日期筛选** | `date:today`, `date:2024-01-01`, `date:>2024-01` | 中 | P1 |
| **路径筛选** | `path:downloads`, `path:C:\Users` | 中 | P1 |
| **通配符支持** | `*` 任意匹配, `?` 单字符 | 中 | P1 |
| **正则搜索** | `regex:\d{4}-\d{2}` 前缀启用 | 高 | P2 |
| **搜索高亮** | 结果中匹配文字标红/加粗 | 低 | P1 |
| **搜索建议** | 输入时下拉提示历史/常用搜索 | 高 | P2 |

**搜索语法示例（目标）：**
```
report pdf              # 同时包含 report 和 pdf（当前已支持）
ext:pdf                # PDF 文件（新增）
size:>10mb             # 大于 10MB（新增，当前只能通过筛选框）
size:<1kb              # 小于 1KB（新增）
size:1mb-100mb         # 1MB 到 100MB 之间（新增）
date:today             # 今天修改（新增）
date:2024-01-01        # 指定日期（新增）
path:downloads         # 在 downloads 目录下（新增）
*.js                   # JS 文件（通配符，新增）
regex:^test_\d{3}$     # 正则匹配（新增）
```

### 2.5 模块四：收藏夹与最近访问（P1 - 重要）

**目标：保存常用文件夹和搜索条件，快速回访**

#### 当前状态
- ❌ 无收藏功能
- ❌ 无最近访问记录
- ❌ 缺少 `file_favorites` 和 `file_access_history` 表

#### 数据库设计（需新建）

```sql
-- 文件收藏表
CREATE TABLE IF NOT EXISTS file_favorites (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  type TEXT NOT NULL CHECK(type IN ('folder', 'search', 'file')),
  name TEXT NOT NULL,
  path TEXT,                    -- 文件夹路径 (type=folder/file)
  query TEXT,                  -- 搜索条件 (type=search)
  icon TEXT DEFAULT 'folder',  -- 自定义图标
  color TEXT DEFAULT '#1976D2', -- 自定义颜色
  sort_order INTEGER DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX IF NOT EXISTS idx_favorites_type ON file_favorites(type);
CREATE INDEX IF NOT EXISTS idx_favorites_created ON file_favorites(created_at);

-- 文件访问历史表
CREATE TABLE IF NOT EXISTS file_access_history (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  file_id INTEGER,             -- 关联 files 表（可为空，如文件夹）
  path TEXT NOT NULL,
  name TEXT NOT NULL,
  access_type TEXT DEFAULT 'open' CHECK(access_type IN ('open', 'preview', 'edit')),
  accessed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (file_id) REFERENCES files(id) ON DELETE SET NULL
);
CREATE INDEX IF NOT EXISTS idx_access_history_file ON file_access_history(file_id);
CREATE INDEX IF NOT EXISTS idx_access_history_time ON file_access_history(accessed_at);
CREATE INDEX IF NOT EXISTS idx_access_history_path ON file_access_history(path);
```

## 五、风险与注意事项

### 5.1 性能风险

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 大结果集渲染卡顿 | 用户体验差 | Phase 2 引入虚拟滚动 |
| 搜索响应慢（>500ms） | 即时搜索失效 | 增加 loading 状态、优化 SQL 索引、考虑缓存 |
| 频繁写入访问历史 | 数据库膨胀 | 异步批量写入、定期清理（保留 30 天） |
| 分类统计查询慢 | 侧边栏加载数据延迟 | 缓存统计结果（5 分钟过期）、增量计算 |

### 5.2 兼容性风险

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 移动端 hover 不适用 | 无法使用预览 | 增加 click 触发预览 |
| 小屏幕侧边栏占用空间 | 内容区太窄 | 收起为抽屉/底部导航 |
| Electron vs 浏览器差异 | 部分 API 不可用 | 环境检测 + 降级处理 |

### 5.3 数据安全

| 注意事项 | 说明 |
|----------|------|
| 访问隐私 | 最近访问记录涉及用户行为，应提供"不记录"选项 |
| 收藏数据 | 收藏夹是用户个人数据，需支持导出/导入 |
| SQL 注入防护 | 所有用户输入必须参数化查询（使用 prepared statement） |

### 8.3 代码规范提醒

#### 组件规范

1. **组件文件结构**
```
src/components/ComponentName/
├── index.tsx             # 组件入口（TSX 语法）
├── index.scss            # 组件样式
└── types.ts              # 可选：类型定义
```

2. **组件命名**
- 文件夹：PascalCase（首字母大写）
- 入口文件：`index.tsx`（不是 `.vue`）
- 样式文件：`index.scss`

#### 样式规范

1. **设计令牌位置**
```
src/styles/
├── design-tokens.scss    # 设计令牌（颜色、间距、字体、阴影等）
├── components.scss       # 通用组件样式
├── transitions.scss     # 过渡动画
└── micro-interactions.scss
```

2. **设计令牌包含内容**
| 系统 | 内容 | 示例 |
|------|------|------|
| 间距 | `$spacing-xs` ~ `$spacing-xxxl` | `$spacing-lg: 16px` |
| 字体 | 字体族、大小、字重 | `$font-size-base: 14px` |
| 颜色 | 主色、功能色、文字色 | `$color-primary: #1976D2` |
| 圆角 | `$radius-sm` ~ `$radius-full` | `$radius-md: 8px` |
| 阴影 | `$shadow-sm` ~ `$shadow-xl` | `$shadow-md` |
| 过渡动画 | 时长、缓动函数 | `$duration-fast: 150ms` |
| Z-index | 层级系统 | `$z-index-modal: 1050` |

3. **样式写法**
```scss
// ✅ 推荐：使用设计令牌
@use '@/styles/design-tokens.scss' as tokens;

.my-component {
  padding: tokens.$spacing-md;
  border-radius: tokens.$radius-md;
  background: rgb(var(--v-theme-surface));
  color: rgb(var(--v-theme-on-surface));
  transition: all tokens.$duration-fast tokens.$ease-default;
}

// ❌ 避免：硬编码
.my-component {
  padding: 12px;
  color: #FFFFFF;
}
```
